# Tools Reference

A tool matters only when you understand the problem boundary it owns. This folder explains what each tool category does, what concept sits behind it, and when it becomes useful in the journey.

Think of tools as the machinery around the idea. The model is not the whole system. Tools decide how the system reads files, calls APIs, stores data, checks results, and stays safe.

## How To Study A Tool

- Identify the problem first.
- Learn the important nouns and control points.
- Try the tool on one small project before judging it.
- Compare it with the simpler alternative. That is where real understanding usually appears.
