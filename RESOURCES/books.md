# Books

This list is not meant to be read front to back in one burst. Each book belongs to a phase for a reason. The goal is just-in-time reading that supports current build work.

## Reading Strategy

- read for leverage, not completion
- extract ideas into the relevant phase file
- build something immediately after a strong chapter
- return later when a concept becomes practical

## Priority Reading Order

| # | Book | Best Phase | Why it matters | How to use it |
| --- | --- | --- | --- | --- |
| 1 | AI Engineering - Chip Huyen | 2 and 7 | Strong systems view of model behavior, evaluation, and production thinking | Read early chapters in Phase 2, come back in Phase 7 for orchestration and reliability |
| 2 | Hands-On Large Language Models - Alammar & Grootendorst | 2 and 3 | Good bridge from model fundamentals to practical LLM workflows | Read selectively while learning APIs and prompt/context patterns |
| 3 | The LLM Engineering Handbook - Iusztin & Labonne | 5 to 7 | Strong coverage of modern LLM application building | Use as a companion once retrieval and orchestration work begins |
| 4 | Building LLMs for Production - Bouchard & Peters | 7 to 9 | Useful for production concerns, evals, and operational thinking | Read once the projects stop being local experiments |
| 5 | Build a Large Language Model (from Scratch) - Raschka | 10 | Best for developing model intuition during the fine-tuning phase | Read selected chapters, not necessarily the whole book |
| 6 | The Hundred-Page Machine Learning Book - Burkov | 1 | Fast intuition for the wider ML landscape | Use as a quick orientation, not the main curriculum |
| 7 | Designing Machine Learning Systems - Chip Huyen | 9 | Helpful for measurement, deployment, and system tradeoffs | Read when evals and observability become real needs |

## Personal Notes

- The most important book here is probably `AI Engineering` because it ties separate topics into one discipline.
- The biggest mistake would be replacing projects with reading. Books should sharpen what the code work is already revealing.
- If time gets tight, prioritize chapters tied to the current phase over "completing" a title.
