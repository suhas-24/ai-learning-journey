# Resources Reference System

This folder is the study companion for the learning journey. It is not a list of links to collect. It is a set of beginner-friendly guides that explain what each resource is, what problem it helps solve, what idea it teaches, when it helps, and what to skip when you are still learning.

If some terms here feel unfamiliar, that is expected. The resource pages define their own vocabulary, but a few words will appear often:

- An `LLM` is a large language model. It reads text and predicts what text should come next.
- A `token` is a small chunk of text the model processes.
- `Tokenization` is the act of breaking text into tokens.
- An `embedding` is a list of numbers that stands for meaning.
- `Retrieval` means finding useful information before the model answers.
- An `eval` is a test that checks whether a system is getting better.
- `Fine-tuning` means training an already-trained model on extra examples so it behaves differently.
- `Orchestration` means coordinating steps, tools, and state so the system follows a clear path.

## How To Use This Layer

- Start with [by-phase.md](./by-phase.md) when you want the shortest path for the phase you are in.
- Use the category guides when you want to understand the kind of resource before choosing a specific title.
- Treat every resource as a tool for a build decision. If it does not help you design, debug, measure, or ship something, it is probably not the best use of your time right now.

## What Each Category Means

- [Books](./books/README.md) explain the ideas slowly and deeply.
- [Courses](./courses/README.md) give you a guided first pass and a bit of practice.
- [Tools](./tools/README.md) show how ideas become systems, workflows, and boundaries.
- [People](./people/README.md) show how experienced builders explain the same ideas in different words.
- [Newsletters](./newsletters/README.md) help you notice changes without turning learning into noise.

## Recommended Reading Order

1. Use [by-phase.md](./by-phase.md) to find the smallest useful set for the phase you are in.
2. Read one concept guide from [books](./books/README.md) or [courses](./courses/README.md).
3. Pair that with the matching [tools](./tools/README.md) guide so you understand what to build or use.
4. Use [people](./people/README.md) and [newsletters](./newsletters/README.md) to strengthen judgment, not to replace practice.

## Fast Start Paths

- If you are new to programming and AI, begin with [books/ml-foundations-and-thinking.md](./books/ml-foundations-and-thinking.md) and [tools/python-foundation-tools.md](./tools/python-foundation-tools.md).
- If you are learning how to talk to models, begin with [courses/prompting-and-api-basics.md](./courses/prompting-and-api-basics.md) and [tools/model-api-tools.md](./tools/model-api-tools.md).
- If you are building retrieval systems, begin with [books/ai-engineering-and-llm-systems.md](./books/ai-engineering-and-llm-systems.md), [courses/rag-and-retrieval-courses.md](./courses/rag-and-retrieval-courses.md), and [tools/retrieval-data-tools.md](./tools/retrieval-data-tools.md).
- If you are learning agents or tool use, begin with [courses/orchestration-and-agents-courses.md](./courses/orchestration-and-agents-courses.md) and [tools/agent-orchestration-tools.md](./tools/agent-orchestration-tools.md).
