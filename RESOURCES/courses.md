# Courses

Courses are support material, not the main engine of progress. Short, focused courses are preferred because they map better to the build-first style of this roadmap.

## How To Use Courses Well

- watch with a project open beside the video
- pause to implement examples instead of bingeing content
- steal structure, not just code snippets
- treat every course as one input into a build, not as completion by itself

| OpenAI Cookbook | OpenAI / GitHub | 2 | Good reference for tools, structured output, and eval patterns | Compare patterns across providers rather than copying blindly |
| W&B AI Academy - MLOps and Evals | Weights & Biases | 9 | Good introduction to disciplined model evaluation | Use it to strengthen observability thinking |

## Personal Notes

- The best "course" might end up being documentation plus one real project.
- DeepLearning.AI short courses are useful because they are fast and focused.
- Cookbooks are especially valuable because they are closer to real implementation than lecture-style content.
