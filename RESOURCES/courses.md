# Courses

Courses are useful when they compress a concept into a short, focused pass. The right course should give me a mental model, a vocabulary set, and a small set of build ideas. It should not become a substitute for the actual project work.

## How To Use Courses

- pair every course with one active project or exercise
- pause when a concept matters enough to test it immediately
- extract the topic list into the relevant phase file
- ignore anything that does not change how I build

## Personal Notes

- The best courses are the ones that force me to build immediately afterward.
- Short courses are more useful than long ones when the goal is actual engineering fluency.
- If a course does not map cleanly to a phase or project, it is probably not the next thing I should study.
