# Newsletters Reference

Newsletters are filters, not a curriculum. Their job is to help you notice meaningful shifts in tools, research, and product patterns without turning your learning process into continuous distraction.

## Guides In This Folder

- [weekly-signal-scanners.md](./weekly-signal-scanners.md)
- [technical-analysis-and-research-interpretation.md](./technical-analysis-and-research-interpretation.md)
- [strategy-policy-and-ecosystem.md](./strategy-policy-and-ecosystem.md)

## How To Use Newsletters Well

- Read on a schedule, not reactively.
- Save only ideas that change a design, evaluation, or tool decision.
- Drop any source that increases noise more than judgment.

For phase-specific suggestions, use [../by-phase.md](../by-phase.md).
