# Newsletters Reference

Newsletters are filters, not a curriculum. Their job is to help you notice meaningful shifts in tools, research, and product patterns without turning learning into constant distraction.

Think of them as a weekly check-in, not your main teacher. If a newsletter makes a concept clearer, keep it. If it only adds noise, skip it.

## How To Use Newsletters Well

- Read on a schedule, not reactively.
- Keep only the ideas that would change a design, evaluation, or tool decision.
- Drop any source that creates more noise than judgment.
