# FAQ

This page answers the questions that usually show up first when the vocabulary is still new.

## What is an LLM?

An LLM is a large language model.

It is software trained on lots of text so it can predict and generate text.

## What is a token?

A token is a small piece of text the model reads or writes.

It is not always a full word.

## What is tokenization?

Tokenization is the step that breaks text into tokens so the model can process it.

## Why does the repo keep talking about tokens instead of words?

Because the model does not work with words the way people do.

It works with tokens, and many limits, costs, and measurements are based on tokens.

## What is a context window?

The context window is the amount of text the model can see at one time.

Think of it as the model’s short-term working space.

## Do I need to finish every chapter before building?

No.

The rhythm should be: learn a concept, build something small, then come back for the next concept.

## Can I skip straight to agents?

You can, but the result is usually brittle.

Without Python, APIs, and debugging habits, later phases feel magical until they break.

## Do I need to know machine learning math deeply?

Not to start.

You need enough intuition to understand what models are good at, where they fail, and how retrieval or evaluation changes system quality.

## When should I use AI coding tools?

Use them as accelerators, not replacements for understanding.

If they write something you cannot explain, slow down and study it.

## How do I know I am ready for the next phase?

You are ready when you can explain the current phase in simple language and build a small version of the main idea.
