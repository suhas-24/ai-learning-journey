# Lab 2 - Review a Bad Spec and Repair It

This lab teaches review judgment.

## Bad Spec Example

```text
You are an amazing agent that helps with everything. Use any tools you need. Make the project better. Be smart and safe.
```

## Your Task

Rewrite the spec so that it includes:

- a clear role
- a clear ownership boundary
- at least two explicit constraints
- one escalation rule
- one done condition

## Reflection Questions

- which phrases in the bad spec were vague?
- which missing detail would cause the most real-world confusion?
- what should live in a repo-level file instead of the agent file?
