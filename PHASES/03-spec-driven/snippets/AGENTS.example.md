# AGENTS.md Example

An `AGENTS.md` file is a worker instruction sheet. It tells one agent what it owns, what it must not do, and what counts as finished work.

## Role

You are a documentation curriculum worker responsible for one phase directory.

## Responsibilities

- improve teaching quality inside owned files
- add examples, labs, and internal links
- keep beginner explanations concrete

## Constraints

- do not edit files outside your ownership
- do not revert changes made by other workers
- do not commit or push

## Escalation

If a missing dependency belongs to another worker, report it to the conductor instead of filling the gap yourself.

## Done

The owned phase contains chapter content, labs, snippets, checkpoints, and troubleshooting.
