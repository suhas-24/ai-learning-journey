# Phase 2 Checkpoints

Use these to test understanding, not memory.

## Concept Checks

1. What is an LLM in plain language?
2. What is a token?
3. What is tokenization?
4. What is a prompt?
5. What is a message?
6. What does the `system` role do?
7. What does `context` mean?
8. What is a tool call, in your own words?

## Practical Checks

- sketch a message array with system, user, assistant, and tool roles
- simulate one tool call by hand
- trim a bloated context window into a focused one
- explain why structured output is easier for code than free-form text
- write down three possible causes of one bad answer

## Exit Standard

You are ready for the next phase when you can explain the full path from user request to model output to tool result to final answer without relying on jargon.
