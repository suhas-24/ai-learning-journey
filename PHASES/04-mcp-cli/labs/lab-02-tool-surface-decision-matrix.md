# Lab 2 - Tool Surface Decision Matrix

In this lab, a `decision matrix` is just a table that helps you choose the best option for a job.

This lab trains the judgment layer, not just implementation skill.

## Instructions

For each task below, choose `CLI`, `API`, `MCP`, or `A2A`, then explain why in two sentences:

1. Run a local test suite after every generated code change.
2. Query a hosted billing service for subscription status.
3. Expose a reusable company-wide tool for incident creation.
4. Ask a deployment specialist agent to validate a rollout plan.
5. Pull exact issue metadata from GitHub in a backend service.

## Scoring Rubric

Your answer is strong if it:

- names the lightest safe option
- mentions auth and environment assumptions
- explains why the rejected options are heavier or weaker

## Optional Extension

Design a matrix with these columns:

| Task | Side effect | Environment | Auth owner | Reuse need | Best tool surface |
| --- | --- | --- | --- | --- | --- |

If you can fill that matrix quickly, your decision framework is becoming operational.
