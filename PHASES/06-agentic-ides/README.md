# Phase 06 - Agentic IDEs, Coding Agents, and Review Workflows

This module teaches how to work with coding agents without surrendering engineering judgment. The goal is not to become dependent on an assistant. The goal is to learn how to scope work, provide context, review outputs, and build a repeatable loop that stays safe under pressure.

## What You Will Learn

- how agentic IDEs, CLI agents, and editor extensions differ in practice
- how to frame tasks so an agent can succeed without guessing
- how to review generated diffs, logs, and tests like an engineer instead of a spectator
- how to split work across subagents without causing file collisions or context loss

## Core Principle

Use agents for speed and coverage. Keep architecture, risk acceptance, and final judgment human-led.

## Suggested Path

1. Read Chapter 1 and classify the coding tools you already use.
2. Read Chapter 2 and adapt [snippets/task-brief-template.md](./snippets/task-brief-template.md) for a real repo task.
3. Read Chapter 3 before you trust any generated diff.
4. Read Chapter 4 if you want to parallelize work with multiple agents safely.
5. Complete the labs and use [checkpoints.md](./checkpoints.md) to pressure-test your workflow.
