# Exercise 2 - Portfolio Review

## Goal

Review each portfolio project as if you were a hiring manager opening it for the first time.

That means noticing whether the project is understandable, believable, and complete without hidden context.

## Review Questions

- Can I understand the problem in under 30 seconds?
- Is the architecture visible?
- Is there evidence that quality was measured?
- Is there a demo or walkthrough?
- Are the limitations honest?
- Would I believe this person could discuss the system in detail?

## Scoring

Rate each project from 1 to 5 on:

- clarity
- engineering depth
- evaluation quality
- demo quality
- credibility

Anything below 4 should trigger a rewrite pass.
