# Portfolio Troubleshooting

When a portfolio feels weak, the fix is usually to add clearer evidence, clearer explanation, or a clearer scope.

## Symptom: The Repo Looks Busy But Not Impressive

Likely cause:

- too much code context and not enough explanation

Fix:

- rewrite the README intro
- add architecture and evaluation sections
- remove vague claims and replace them with specifics

## Symptom: I Do Not Know What To Show In A Demo

Likely cause:

- the project lacks a defined user flow or scorecard

Fix:

- script one representative scenario
- add one metric and one limitation to the demo

## Symptom: Resume Bullets Feel Generic

Likely cause:

- they describe tools used instead of outcomes and proof

Fix:

- include system type, mechanism, and evaluation evidence in the same sentence

## Symptom: I Freeze When Asked About Tradeoffs

Likely cause:

- you have not written down architecture choices and rejected alternatives

Fix:

- add a "why this design" section per project
- note one thing you intentionally did not build

## Symptom: The Portfolio Sounds Like Coursework

Likely cause:

- too much emphasis on learning goals and not enough on shipped artifacts

Fix:

- lead with the problem, system, evidence, and limitation
- keep "what I learned" as a short retrospective, not the main story
