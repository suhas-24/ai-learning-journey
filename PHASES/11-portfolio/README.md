# Phase 11 - Portfolio And Job Readiness

This phase turns your private competence into public proof. The goal is not to decorate GitHub. The goal is to make another engineer believe, within a few minutes, that you can build, evaluate, and communicate production-minded AI systems.

## Outcomes

By the end of this phase, you should have:

- three portfolio-grade projects with clear READMEs, evals, and demos
- a concise personal narrative connecting those projects into one engineering story
- resume bullets grounded in shipped work, not generic buzzwords
- interview answers for architecture, tradeoffs, failure analysis, and metrics
- a repeatable weekly pipeline for portfolio polish and job outreach

## What This Phase Teaches

Portfolio quality is a technical skill. You are learning to:

- scope evidence that proves engineering depth
- explain architecture without rambling
- present metrics honestly
- show failure handling, not only best-case demos
- translate project work into resume, GitHub, and interview language

## Portfolio Standard

Every major project should make these five things obvious:

1. What problem you solved
2. How the system works
3. How you evaluated it
4. What failed and what you changed
5. Why your design choices were reasonable

If a stranger cannot answer those five questions from your repo, the project is not finished.

## The Three Required Proof Types

- Build proof: code, architecture, folder structure, setup
- Quality proof: evals, metrics, error analysis, test artifacts
- Communication proof: README, diagrams, demo, interview-ready summary

## Time Budget

- Week 1: define portfolio narrative and pick strongest project evidence
- Week 2: improve READMEs, diagrams, screenshots, demos
- Week 3: write eval summaries and failure stories
- Week 4: convert project evidence into resume, LinkedIn, and outreach assets

## Final Deliverable

Create a portfolio packet for each project that includes:

- one polished README
- one architecture diagram or equivalent explanation
- one evaluation summary
- one short demo
- one set of interview bullets
